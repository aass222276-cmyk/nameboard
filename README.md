# Manga Name Editor (v3)

ネーム（下書き）段階の“テキストとコマ割り”を最速で決めるための極小エディタ。
GitHub Pagesで動作するPWA（プログレッシブ・ウェブアプリ）です。

## 特徴

* **高速な入力**: クリックでフキダシを即作成し、入力できます。
* **改行の維持**: テキストは自動で折り返されず、`Enter`キーによる改行のみが反映されます。
* **コマ割り**: ドラッグの角度で水平/垂直を判定し、対象のコマ「だけ」を分割します。
* **入出力**: テキスト形式でのコピー/ペーストに対応し、他ツールと連携できます。
* **書き出し**: 現在のページをPNG、全ページをZIPで書き出せます。
* **ローカル保存**: データはすべてお使いのブラウザ内 (localStorage) に保存され、サーバーには一切送信されません。
* **オフライン対応**: PWAに対応しており、オフラインでも動作します。

## セットアップ (GitHub Pages)

1.  このリポジトリをフォークまたはご自身のアカウントにコードをプッシュします。
2.  **`icons` フォルダ**を作成し、以下の2つのアイコン画像（PNG形式）を配置します。
    * `icons/icon-192.png` (192x192ピクセル)
    * `icons/icon-512.png` (512x512ピクセル)
3.  リポジトリの **Settings** > **Pages** を開きます。
4.  **Source** を **Deploy from a branch** に設定します。
5.  **Branch** を `main` / `(root)` に設定し、**Save** します。
6.  デプロイが完了すると表示されるURLにアクセスします。

## キャッシュの更新について

ファイルの変更が反映されない場合（特に `app.js` の更新後）、Service Workerのキャッシュが原因である可能性があります。
その場合は、`sw.js` ファイル内の `SW_VERSION` の数値を1つ上げてください。

例:
`const SW_VERSION = 'manganame-v3.0.0';`
↓
`const SW_VERSION = 'manganame-v3.0.1';`

変更をコミット＆プッシュすると、次にアクセスした際に新しいバージョンがインストールされます。
